如何用Hexo + Github Page快速构建个人博客


Github Page可以理解为一个服务器，Hexo可以理解会博客的样式外壳。
用Hexo + Github Page构建个人博客首先要在GitHub上获得一个特殊的资源池。

初始化Github Page

按照Github Page官方网站的5个步骤，就可以在对应的http://username.github.io下访问自己的博客了。
但是这个博客非常的简单原始，所以需要用一个比较漂亮的主题给自己的博客穿上新衣。

如果对自己的博客要求不是很多，只要静态网站就可以满足的话，建议使用Hexo或Jeklly。
外貌协会在决定前先过了一遍这两个博客平台的主题们，感觉Hexo看着更舒服一些，就决定用Hexo了。

安装Hexo

在安装Hexo之前，先确保node.js的环境。

安装主题

主题的选择有很多，也不用过分追求主题，沉迷其中无法自拔，还是要选个实用的。比较了好多，最终决定在只写博客的情况下还是用默认的好。安装主题很简单，在https://hexo.io/themes/中有很多可供选择，可预览，而且每个主题的链接跳转到相应的github里，都介绍了具体的主题安装使用及配置说明。
以下使用Landscape主题为例子

修改Hexo目录下的_config.yml配置文件中的theme属性，将其设置为landscape。
如果需要对landscape进行设置，themes/landscape 下可在_config.yml中进行配置。

将Hexo部署到Github Page

前面两步都很简单，按部就班即可。如何将本地的Hexo部署到Github Page上呢？是不是将整个项目push到Github上就可以了呢？
在前面初始化Github Page的过程是，如果将index.html文件换个名字叫做homepage.html可以吗？
事实是不可以的。因为index.html文件是Github Page访问的入口文件。而直接将Hexo项目push到Github上必然会导致Github Page找不到入口文件从而会产生404错误。

借力前辈的经验和Hexo文档，发现Hexo有generate和deploy（翻译成“生成”和“部署”吧）两个概念。

第一次运行hexo generate命令会在当前目录下生成一个新的叫做public的文件夹。内部的文件夹结构为

  • 2017
    • …
  • archives
    • …
  • css
    • …
  • fancybox
    • …
  • js
    • …
  • index.html

正是Github Page所需要的。所以将生成的public文件夹下的内容上传即可。
在创建的blog文件夹下clone初始化好的Github Page。

但是这一部分的命令相当多，操作起来不是那么方便，可以放到一个shell文件里。

在Mac中执行上述文件的过程中可能会出现Permission denied的情况。这时用命令修改该文件的操作权限即可。

在不知道脚本执行的具体内容的情况下，千万不要乱修改权限哦，会出大事的。
